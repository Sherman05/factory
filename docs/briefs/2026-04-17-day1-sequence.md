# Day 1 Sequence — Phone-first control loop

**Цель дня:** к концу дня с телефона жать одну кнопку "Merge" на готовые PR и командой `/new <описание>` заводить новые задачи.

**Три brief'а, в строгом порядке** (каждый — отдельная задача в vibe-kanban, каждая ≤2ч агентского времени):

## Порядок

1. **Brief A — `/new <description>`** (`brief-A-telegram-new-command.md`)
   Команда в Telegram записывает brief-файл в репо. Не зависит от B и C.
   → После merge: можно в чате `/new "Сделай X"` → файл коммитится → ты копируешь путь в vibe-kanban.

2. **Brief B — PR watcher** (`brief-B-pr-watcher.md`)
   Бот раз в 2 минуты опрашивает GitHub API, кидает тебе уведомление о новом PR.
   → После merge: забываешь про браузер, узнаёшь о готовых PR в телеграме.

3. **Brief C — Inline Approve/Merge кнопки** (`brief-C-inline-merge.md`)
   Зависит от B (встраивается в уведомление оттуда).
   → После merge: один тап с телефона мержит PR в main.

## Почему именно такой порядок

- A не зависит от B/C — можно запустить первым.
- B даёт событие "PR готов" — на нём висит C.
- C без B бессмыслен (нет точки входа для кнопки).

## Как запускать

Для каждого brief'а:
1. Открыть vibe-kanban (не забыть `$env:HTTPS_PROXY` перед запуском!).
2. Новая задача → в описание: `Brief: docs/briefs/<file>.md. Прочитай и реализуй по acceptance criteria.`
3. Run → смотришь → ревью PR → Merge (или с телефона через /notify, когда B+C готовы).

## Общие правила для всех трёх

- Расширяем существующий `products/telegram-control-bot/`, **не создаём новый продукт**.
- Prefer small units — каждая новая фича в своём файле `src/*.ts` (≤100 строк).
- TDD обязательно (как в предыдущей задаче).
- **Proxy-совместимость:** любой новый outbound HTTP вызов (к GitHub API, Telegram API) должен идти через `HTTPS_PROXY` — паттерн уже есть в `src/bot.ts`.

## Вне scope Day 1

- Прямое создание задач в vibe-kanban (его API не документирован → Day 2).
- Статус "в процессе" от vibe-kanban → Day 2.
- Деплой бота на Cloudflare → Day 3.
- Multi-user whitelist — не нужно для личного бота.
